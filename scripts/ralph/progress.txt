## Codebase Patterns
- SimulationStepCache model in `frontend_server/translator/models.py` stores per-step, per-stage data keyed by (simulation, step, stage). Stages: PERCEIVE, RETRIEVE, PLAN, REFLECT, EXECUTE.
- `CanObserveSimulation` DRF permission class already defined in `frontend_server/translator/api_views.py` — use it on all simulation-scoped read endpoints.
- Frontend uses `retro-*` Tailwind utility classes from `design-system/colors_and_type.css` for all UI components.
- API client pattern: authenticated fetch wrapper in `frontend/src/api/simulations.ts` — follow existing function signatures.
- SimulatePage `meta.step` is the authoritative current step; SSE events update it via `useSimulationSSE` hook.
- Agent movement/action data arrives via SSE (`useSimulationSSE`) as `StepUpdatePayload` with `persona_movements` field.
