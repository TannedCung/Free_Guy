## Codebase Patterns
- Project uses mypy for typechecking, ruff for linting — run via pre-commit hooks on commit
- `backend_server/pyproject.toml` configures mypy and ruff for the backend
- `backend_server/requirements.txt` and `frontend_server/requirements.txt` track Python deps
- Django version differs: backend uses 2.2, frontend uses 4.2.20
- Use `pre-commit run --files <file>` to typecheck/lint individual files before committing
- Qdrant point IDs should be deterministic UUIDs (uuid5) derived from (persona_id, embedding_key) to make upserts idempotent
- `_ensure_collection()` pattern: check existing collections set before creating to avoid errors on repeated calls
- Module-level `_client: Optional[QdrantClient] = None` + lazy init in `_get_client()` is the singleton pattern used
